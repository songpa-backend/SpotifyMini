## 프로젝트 설명
- SpotifyMini는 사용자가 음악을 검색하고 좋아요를 한 곡을 모아 나만의 플레이리스트를 만들 수 있는 미니 음악 서비스입니다. <br>
- 사용자가 처음 화면 접속 시, 플레이리스트를 표시합니다. 곡명 또는 가수명으로 음악을 검색 시 검색한 음악을 목록에 표시합니다. 검색 결과가 없는 경우 검색 결과 없음을 표시 합니다.<br>
- 곡 목록/곡 상세정보에서 좋아요(♥) 를 클릭하고, 좋아요 누른 노래는 ♡/♥ (빈하트/꽉찬 하트)로 상태가 표시되고, 좋아요(♥) 상태의 노래는 '내 좋아요 목록'에서 따로 확인이 가능합니다. <br>
- 곡 제목 옆 '상세보기'를 클릭하여 각 개별 곡의 상세정보를 확인 할 수 있습니다. 상세 정보에서는 제목,가수,장르, 길이가 표시되고, 댓글창이 표시됩니다.
- 상세보기 화면 하단 댓글 기능으로 해당 곡에 대한 의견을 사용자가 댓글 형식으로 달고, 내용을 다른 사용자와 공유 할 수 있수 있습니다.<br>

## **주요 기능**

## 아키텍처 개요
- SpotifyMini는 전역 상태 관리를 위해 Zustand Store를 사용합니다.
- 노래 목록 데이터와 좋아요(♥) 상태를 Store에서 관리하여, 여러 페이지/컴포넌트에서 동일한 데이터를 공유하고 동기화할 수 있도록 설계했습니다.
### Zustand 스토어 목록
- `useMusicStore`
  - 전체 노래 데이터를 한번만 호출하여 저장
  - 검색/플레이리스트 등 다양한 화면에서 동일한 음악 데이터를 재사용
- `useLikeStore`
  - 좋아요(♥) 상태 동기화 및 찜 목록 관리
  - 좋아요 추가/삭제 시 다른 화면에서도 상태가 동일하게 반영되도록 동기화      

### API 데이터 구조 (db.JSON)
SpotifyMini는 json-server 기반의 Mock API를 사용하며, 데이터 구조는 다음과 같습니다.
- users : 사용자 정보
- musics : 음악 데이터
- comments : 특정 음악에 대한 댓글 (musicId, userId로 연결)
- favorites : 좋아요(찜) 데이터 (userId, musicId로 관계 표현)

```json
{
  "users": [
    { "id": 1, "name": "채린" },
    { "id": 2, "name": "해원" }
  ],
  "musics": [
    { "id": 1, "title": "Magnetic", "artist": "ILLIT", "genre": "K-Pop", "duration": "3:24"}
  ],
  "comments": [
    { 
      "id": 1, 
      "musicId": 1, 
      "userId": 1, 
      "content": "이 노래 너무 중독성 있어요!" 
    }
  ],
  "favorites": [
    { "id": 1, "userId": 1, "musicId": 1 },
    { "id": 2, "userId": 2, "musicId": 1 }
  ]
}
```
